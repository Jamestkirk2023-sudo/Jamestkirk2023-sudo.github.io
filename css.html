Ruleset Terms

  • Selector: The begining of the rule set used to target the element that will be styled.
  • Declaration Block: The code in-between (and including) the curly brace that contains the CSS declarations.
  • Property: The first part of the declaration that signifies what visual characteristic of the element is to be modified.
  • Value: The second part of the declaration that signigies the value of the property.

Example of syntax

                    p {
                    color:blue;
                    }
  • p = Selector
  • area inside of {} = Declaration Block
  • color: blue; = Declaration
  • color = property
  • blue = value

Psuedo Classes

Actions that can be assigned to attributes causing different actions based on declaration passed in.

  • p:hover - Affects an element when cursors hovers over
  • :active - Affects an element when the user interacts with it.

Example

                    p:hover {
                    background-color: lime;
                    }

Descendant Combinator

In addition to chaining selectors to select elements, CSS also supports selecting elements that are nested within other HTML elements. These are also known as descendants.

Consider the following: 

                    <ul class="main-list">
                    <li> ... </li>
                    <li> ... </li>
                    <li> ... </li>
                    </ul>

The nested li elements are descendants of the ul element and can be selected like so:

                    .main_list li {

                    }

The above example shows, .main_list selects the element with the .main-list class(the ul element).
The descendant li elements are selected by adding the li to the selector, separated by a space.
